教程：IDE中配置注释模板 && 使用注释模板生成注释[Top]

概述：开发人员在项目建设过程中，应养成良好的为代码添加合理的注释的习惯，良好的注释有利于提高程序的可读性、易理解性和可维护性。为了帮助开发人员完成代码注释工作，特参照相关资料和实际使用场景，制定了一系列“注释模板”，并编撰出本教程帮助项目组相关开发人员配置并使用相关注释模板完成代码注解工作。

step1, 配置集成开发工具[Top]

step1.1, IDE中配置全局变量${user}[Top]

  • 菜单栏 -> Help -> Edit Custom VM Options -> 添加如下一行： -Duser.name=authorNameX

备注：请将authorNameX替换成开发者自己的名字，名字尽量不要使用中文，否则会出现乱码

step1.2, IDE中导入配置文件(注释模板)[Top]

  • 点击下载配置文件
  • IDE中导入上述配置文件：
    • WebStorm按照如下方式导入：File -> Import Settings ->
    • Android Studio按照如下方式导入：File -> Manage IDE Settings -> Import Settings ->
  • 继续如下操作

step1.3, 重启IDEA，使得配置生效[Top]

  • IDE中修改配置之后，需要重启，重启之后新的配置才会生效。
  • 如果您已经重启，请忽略，否则请先重启IDE，重启之后再进行后续操作。

标准注释模板[Top]
  • 编辑器中输入--hp -> 按“回车”键，即可自动生成如下待办事项注释模板
  • 完善自动生成的注释模板：
    • 完成模板中列出的TODO事项，主要是添加“功能描述”
    • 如果你不是该文件（vue/html...）的创建者，请添加“修改记录”，记录修改者。修改时间、版本号、修改内容
    • 如果你的组件是公共组件，有两个或两个以上的父组件中使用到了你的组件，或者你正在使用一个公共组件，请完善公共组件的头部注释
      • @reference: 组件引用记录。 如果您在其他文件中引用了公共组件，请在@reference中添加相应引用记录，记录引用路径、引用创建时间、引用创建者等相关信息
      • @instructions: 组件使用说明。请在此处添加当前组件的使用注意事项、该组件的对外接口说明等

给“抽象方法/接口方法”添加方法注释[Top]

  • 编辑器中输入/ma -> 按“回车”键，即可自动生成如下待办事项注释模板
  • 完善自动生成的注释模板：
    • 完成模板中列出的TODO事项，包括添加“功能描述”、“注意事项”、“调用说明”
    • 如果你不是该函数的创建者，请添加“修改记录”，记录修改者。修改时间、版本号、修改内容
    • 完善“函数参数”、“返回值”、“异常抛出”注释
  • 备注：依据《阿里java编程规范》，抽象方法 && 接口方法中的注释和普通方法中的注释略有不同，抽象方法和接口方法的注释中，除了返回值、参数、异常说明外，还必须指出该方法做什么事情，实现什么功能，对子类的实现要求，以及调用注意事项。所以我们专门为“抽象方法（包括接口方法）”定义了独特的“注释模板/ma”,请开发者注意区分，不要和普通方法的注释模板混用
